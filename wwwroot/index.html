authFetch() - Auto Refresh Flow

JS: authFetch("/api/weather") │ ├─► fetch("/api/weather", { Authorization: Bearer <accessToken> }) │ │ │ ├─► 200 OK → return data (done!) │ │ │ └─► 401 Unauthorized (token expired) │ │ │ ▼ ├─► fetch("/Authentication/refresh", { credentials: "include" }) │ Browser sends HttpOnly cookie automatically (JS never touches it) │ │ │ ├─► 200 OK → new accessToken in body + new refreshToken cookie (rotation) │ │ │ │ │ ▼ │ │ RETRY original request with new accessToken → 200 OK │ │ │ └─► 401 FAIL → refresh token also expired/revoked → force logout │ └─► Why doesn't the browser do this automatically? Browser only auto-sends cookies. It does NOT understand JWT or "token refresh". This logic lives in JavaScript (authFetch wrapper / Axios interceptor / etc.)

Refresh Token (HttpOnly Cookie)

HttpOnly cookie cannot be read by JavaScript!

To see the refresh token cookie:
1. Open DevTools (F12)
2. Go to Application tab
3. Expand Cookies in the left sidebar
4. Click on https://localhost:6011
5. Look for refreshToken cookie

Properties to observe:
- HttpOnly: true (JS cannot access)
- Secure: true (HTTPS only)
- SameSite: Lax (CSRF protection)
- Path: /Authentication (sent to all auth endpoints)
